Build with pure CSS

CSS Modal is build out of pure CSS. JavaScript is only for sugar. This makes them perfectly accessible.

Optimized for mobile

The modals are designed using responsive web design. They work on all screen sizes from small a mobile to high resolution screens.

Use as Sass plugin

You can use CSS Modal as Sass plugin and apply it to your custom classes. No need to understand all the code.

How to use

Markup

You need to include the markup and content for modals in you website. This has a positive effect on SEO. The example code:

<section class="semantic-content" id="modal-text">
    <div class="modal-inner">
        <header><!-- Header --></header>
        <div class="modal-content"><!-- The modals content --></div>
        <footer><!-- Footer --></footer>
    </div>

    <a href="#!" class="modal-close" title="Close this modal">×</a>
</section>

Using header and footer is optional. Just leave out the tags if you don't want them in a modal.

You should leave the link's href attribtue to close the modal that way in order to prevent the page from scrolling to the top when clicking it.

Sass

If you use Sass you can use the file _modal.scss and include it into your project. Here is an example:

@import "modules/modal";

CSS Modals uses Sass 3.2's placeholders. You can use them by calling them via @extend in your modal's class name. The snippet looks something like that:

.my-awesome-class-name {
    @extend %modal;
}

CSS

If you are not into Sass you can still use Vanilla CSS. Include it in your CSS and don't forget to add it to your build. Here is the snippet for development:

<link rel="stylesheet" href="css/modal.css">

You can now add the markup to your website as stated above and apply a class .modal to the element.

JavaScript

As stated above you don't need JavaScript to get a good experience out of CSS Modals. But there are two issues where JavaScript helps:

  • Pressing escape: If you press ESC on your keyboard while the modal is visible it closes itself. This behavior cannot be done with CSS only.
  • Preventing background page from scrolling: If you scroll within the modal and you reach the end you don't want the page in the background to scroll. To prevent this JavaScript pushs a CSS class selector to the body element.

To add this behavior to your website, please include the JavaScript file modal.js right before the closing body-tag:

<script src="js/modal.js"></script>

Browser Support

This modal is designed to work on all moderne browsers. Unfortunately this does not include Internet Explorer 7 or lower. But we deal with IE 8 — well,… at least it works.

On mobile Safari for iOS and Android 2.3, 4+ is tested, while Android 2.3 has some problems but it works.

In numbers: Chrome, Firefox, Safari 6, Opera 12, IE8 (functional), IE9+.

Modals build with pure CSS

There are a lot of possiblilities to include content in the modal. Actually it is possible to add every bit of HTML to your source-code that you want to. Some examples could be normal text, images, videos (see the other example), iframes which embed services like a contact form and so.

Modals consist of an optional header and footer and a main content area. The main content area is scrollable if the modal's content is higher than the screen has space for.

You can easily deep link to a modal by including the element's id as a hash in the URL.

On drublic.de you can see the modals in action for normal text content and using it with third party services.
